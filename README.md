# transportation-problem
MVVM with my vision of Clean Architecture; Room, RxJava, Koin, Lottie; The solution of the transport problem by the potential method

**Следующий текст взят из ВКР.**
Второй пункт четвертой главы: **Разработка пользовательского интерфейса программы**

В приложении 4 главных экрана: «Поставщики», «Потребители», «Затраты» и «Решение». И 2 вспомогательных экрана: «О приложении» и «История». 
Осуществлять переход между главными экранами можно с помощью выбора нужного пользователю пункта в нижнем меню навигации, а также переход достигается перелистыванием экранов.
Запуск программы начинается с открытия экрана «Поставщики». На данном экране пользователь может добавить поставщика и назначить ему его запас.

<img src="https://i.ibb.co/d68VDzk/Screenshot-20200321-202900-com-ainullov-kamil-transportation-problem.jpg" width="300" height="600">

Перейдя к экрану «Потребители», мы можем добавить потребителей в список и назначить им потребности.

<img src="https://i.ibb.co/6DXhTnC/Screenshot-20200321-202903-com-ainullov-kamil-transportation-problem.jpg" width="300" height="600">

В приложение добавлены динамически изменяющиеся подсказки для того, чтобы пользователю было легче ориентироваться.
Следующим идет экран «Затраты», здесь отобразится матрица затрат, размеры которой будут рассчитываться на основе числа поставщиков и потребителей.

<img src="https://i.ibb.co/Nyn2h9n/Screenshot-20200321-202905-com-ainullov-kamil-transportation-problem.jpg" width="300" height="600">

Последним среди 4 главных экранов идет экран «Решение». 
Решение получится получить, нажав на кнопку «Решить», отцентрированную сверху экрана. Однако перед этим необходимо заполнить предыдущие экраны: «Поставщики», «Потребители», «Затраты».
 
<img src="https://i.ibb.co/TLWXTJc/Screenshot-20200321-202907-com-ainullov-kamil-transportation-problem.jpg" width="300" height="600">

При нажатии на круглую красную кнопку добавления поставщика в экране «Поставщики» пользователь может добавить поставщика. Откроется диалоговое окно, в котором пользователь может назначить запас поставщика. 

<img src="https://i.ibb.co/Rh4msbz/Screenshot-20200322-001035-com-ainullov-kamil-transportation-problem.jpg" width="300" height="600">

После нажатия на кнопку добавить поставщик добавиться в список с элементов. Пользователь может нажать на кнопку-изображение «крестик», чтобы удалить элемент и также можно «смахнуть» элемент с экрана для его удаления.

<img src="https://i.ibb.co/c61csH3/Screenshot-20200321-202921-com-ainullov-kamil-transportation-problem.jpg" width="300" height="600">

Экран «Потребители» работает аналогично экрану «Поставщики».
Если в списке несколько элементов, то при удержании элемента слева можно менять его местами с другими элементами в списке.

<img src="https://i.ibb.co/1XR0pjR/Screenshot-20200321-202931-com-ainullov-kamil-transportation-problem.jpg" width="300" height="600">

Если экраны «Поставщики» и «Потребители» содержат хотя бы по 1 элементу, то на экране «Затраты» отобразится матрица затрат с количеством столбцов равным количеству потребителей и количеством строк равным числу поставщиков. На данном экране можно назначить затраты на грузоперевозку 1 единицы товара.

<img src="https://i.ibb.co/s1DQNsZ/Screenshot-20200321-203003-com-ainullov-kamil-transportation-problem.jpg" width="300" height="600">

При нажатии на кнопку «Решить» произойдет вычисление оптимального плана. Отобразятся минимальные затраты. Также отобразится граф, на котором будет отображены поставщики и потребители, связи между ними. Полученное изображение графа можно увеличивать или уменьшать по желанию пользователя. При нажатии на один из узлов графа на экране всплывет сообщение с детальной информации о перевозках.

<img src="https://i.ibb.co/hVKw1GM/Screenshot-20200321-203111-com-ainullov-kamil-transportation-problem.jpg" width="300" height="600">

Если информация не умещается на экране, то любой экран в приложении можно проматывать вниз. На экране с результатом вычислений внизу экрана предоставлено более детальное текстовое описание о перевозках. Данный текст, при необходимости, можно выделить, скопировать и отправить в другое приложение.

<img src="https://i.ibb.co/R73GGd1/Screenshot-20200321-203114-com-ainullov-kamil-transportation-problem.jpg" width="300" height="600">

В случае необходимости, пользователь может сбросить условия задачи. Для этого ему необходимо на любом из 4 главных экранах нажать на три точки в верхней панели, это приведет к появлению всплывающего меню. Также пользователь может через это всплывающее меню выйти из приложения или же зайти в раздел «О приложении».

<img src="https://i.ibb.co/GkKtB3R/Screenshot-20200321-203027-com-ainullov-kamil-transportation-problem.jpg" width="300" height="600">

При переходе в экран «О приложении» пользователь может узнать для каких целей создано это приложение и как им пользоваться. Также если подсказки мешают, то в этом экране можно осуществить их отключение. Помимо прочего в верхней панели есть кнопка «Написать» и пользователь может связаться с разработчиком, нажав на нее

<img src="https://i.ibb.co/zSw8QcG/Screenshot-20200321-203031-com-ainullov-kamil-transportation-problem.jpg" width="300" height="600">

Последним неописанным ранее экраном является экран «История». Зайдя в историю пользователю будет представлен список всех ранее решенных задач. Нажав на элемент в списке можно загрузить данную задачу и увидеть более детальное описание решение. Также он может взять одну из раннее решенных задач в качестве чертежа, подкорректировать данные задачи и перевыполнив вычисления. В историю добавляют все задачи, после произведения вычислений в экране «Решение». Из списка с решенными задачами на экране «История» пользователь может удалять элементы путем нажатия на кнопку-изображение «крестик», а также можно «смахнуть» элемент с экрана для его удаления.

<img src="https://i.ibb.co/MDTk8Nz/Screenshot-20200321-203042-com-ainullov-kamil-transportation-problem.jpg" width="300" height="600">
